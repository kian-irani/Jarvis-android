# ADR-009 — Universal Language Support Architecture

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Primary user is Persian/English bilingual; the product targets universal language support (v16). RTL and non-Latin scripts must be first-class.

## Decision
Language-agnostic core: all user text treated as opaque UTF-8; multilingual embedding model (paraphrase-multilingual-MiniLM-L12-v2, 50+ languages incl. Persian) for memory/search; LLM prompts carry a language hint detected per message (not per session); UI strings via standard Android resources with RTL-safe Compose layouts; no language-specific branches in business logic.

## Consequences
+ Persian and mixed-language memory search works out of the box (multilingual vectors)
+ Adding a language = translations only, no code
− Multilingual embedding model is larger than English-only (420MB, within ADR's 2GB budget)

## Alternatives considered
English-pivot translation layer (latency, quality loss); per-language models (storage explosion on mobile).
