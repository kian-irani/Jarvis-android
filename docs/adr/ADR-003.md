# ADR-003 — CRDT for Cross-Device Sync

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Brain-Lite (phone, Room/SQLite) and Brain-Full (VPS, PostgreSQL) must sync memories/nodes/tasks while either side can be offline for long periods (Offline Mode, ADR-008).

## Decision
CRDTs for conflict-free merge: LWW-register per column for memories metadata, OR-set for node registry, append-only log for events. Sync is state-based delta exchange over the VISN protocol; schema mirrored between Room and PostgreSQL (same table/column names) so deltas map 1:1.

## Consequences
+ No central lock, offline-first by construction, deterministic convergence
− LWW loses concurrent edits to the same field (acceptable: single-user system); tombstone growth needs periodic compaction

## Alternatives considered
Server-wins sync (loses offline edits); operational transform (complexity unjustified without collaborative editing).
