# ADR-011 — Privacy Threat Monitor Policy

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Vision's promise is sovereign intelligence: user data must not leak to third parties, including via our own cloud LLM calls.

## Decision
A Privacy Threat Monitor inspects every outbound boundary: (1) cloud LLM prompts pass a PII scrubber (configurable allow-list; raw secrets/tokens always blocked); (2) telemetry is opt-in and local-first; (3) all outbound destinations must be on a user-visible registry — anything else is blocked and logged; (4) per-conversation "local-only" flag forces Offline-mode handling (ADR-008) for sensitive topics.

## Consequences
+ Leak surface is enumerable and user-auditable (registry + logs)
+ Aligns product with its core brand claim
− Scrubbing adds latency to /chat; allow-list maintenance is a real cost

## Alternatives considered
Trust-the-provider (contradicts product thesis); full local-only (kills capability until local LLMs mature on phones).
