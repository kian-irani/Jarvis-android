# ADR-005 — LZ4+zstd for VISN File Transfer Protocol

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
VISN protocol moves files/embeddings between mesh nodes over LAN or constrained uplinks; phones are battery/CPU limited.

## Decision
Two-tier compression: LZ4 for interactive/small payloads (<1MB, latency-sensitive), zstd level 3 for bulk transfer (model files, DB deltas). Negotiated per-transfer in the VISN handshake; raw passthrough for already-compressed formats (apk, onnx, jpg).

## Consequences
+ LZ4 keeps interactive latency negligible on mobile CPUs; zstd ~3x better ratio for bulk
− Two codecs in every node implementation (Kotlin + Python); mitigated by shared conformance test vectors in sdk/

## Alternatives considered
gzip only (worse ratio and speed); brotli (slow compression on mobile); no compression (LAN fine, WAN painful).
