# ADR-006 — Brain Score Algorithm for Auto-Election

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Multiple devices can host a Brain. The mesh must deterministically pick the primary without a coordinator.

## Decision
Each node computes a local Brain Score: base by class (VPS 100 / desktop 80 / phone 50) + uptime and resource bonuses − penalties (battery −20, thermal throttling −30). Highest score wins; ties broken by node id. User override always wins. Auto-failover: when primary misses 3 heartbeats, next-highest takes over (ADR-003 makes state transfer safe).

## Consequences
+ No election protocol complexity (scores are locally computable and gossiped with heartbeats)
+ Predictable, explainable choice (Brain Election UI shows the math)
− Score weights are heuristic; tuning tracked as follow-up after real-device telemetry

## Alternatives considered
Raft leader election (heavy for ≤10 nodes, needs quorum — phones sleep); manual-only selection (poor UX on failover).
