# ADR-007 — Trust Level System for Agent Permissions

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Agents (M3) will read files, send messages, and execute commands. A single all-or-nothing permission is dangerous.

## Decision
Four trust levels per agent: T0 read-only sandbox → T1 + app-private writes → T2 + external reads (web, APIs) → T3 + irreversible actions (send/delete/spend), T3 always requires per-action user confirmation. Levels enforced at tool-node boundary (ADR-004), persisted per agent, downgradable anytime. Anomaly detection (ADR-010) can force-downgrade.

## Consequences
+ Blast radius bounded by level; audit log per tool call
− Friction on T3 actions is deliberate; batch-approval UX needed later

## Alternatives considered
Per-tool ACLs only (no graduated path); OS-style capability tokens (overkill for single-user v1).
