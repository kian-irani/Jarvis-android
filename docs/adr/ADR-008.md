# ADR-008 — Offline Mode Degradation Policy

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Vision must stay useful with no VPS and no internet (ADR-003 enables data convergence afterwards).

## Decision
Three modes with automatic transitions: Online (Brain-Full primary, cloud LLMs) → Hybrid (Brain-Lite primary, cloud LLM via phone data, sync when possible) → Offline (Brain-Lite only: local embeddings/search/memory work; /chat returns explicit DEGRADED_OFFLINE with queued-prompt option, optional local LLM later). Every capability declares its per-mode behavior in /health capabilities; UI surfaces current mode.

## Consequences
+ No silent failures — degradation is explicit and queryable
+ Queued prompts replay on reconnect
− Capability matrix must be maintained as features land (CI check planned)

## Alternatives considered
Hard-fail offline (unacceptable for a sovereign-intelligence product); transparent best-effort (silent quality drops erode trust).
