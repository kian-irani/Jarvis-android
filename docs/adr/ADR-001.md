# ADR-001 — Clean Architecture + MVI for Android

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
The Android app hosts UI (Compose), a local HTTP server (Brain-Lite), and mesh logic. Without clear layering these concerns tangle quickly, and the app must stay testable on JVM (no device farm).

## Decision
Clean Architecture with MVI for UI state. Layers: `ui` (Compose + ViewModels, MVI reducers) → `domain` (use-cases where logic is non-trivial) → `data` (repositories; Room/HTTP/ONNX behind interfaces). Brain-Lite routes depend on ports (`ChatPort`, `EmbedPort`), never concrete repos. Hilt for DI.

## Consequences
+ JVM-testable layers (proven: 16 brain-lite tests run without a device)
+ Mesh slice can swap repository implementations without touching routes/UI
− More files/boilerplate than a flat structure; use-case layer added only when logic warrants (pragmatic, see brain-lite which skips it for CRUD)

## Alternatives considered
Flat monolith (fast, untestable); full modularized Clean Arch with separate Gradle modules (premature while app is small).
