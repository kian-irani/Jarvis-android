# ADR-002 — Hexagonal Architecture for Brain-Full

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
Brain-Full (Python/FastAPI on VPS) integrates many externals: LLM providers, Telegram, DBs, workflow engine. Provider churn is high (key rotation, new APIs).

## Decision
Hexagonal (ports & adapters): domain core with ports; adapters for FastAPI (inbound), PostgreSQL/SQLAlchemy, LLM providers, Temporal (outbound). Pydantic v2 models at boundaries only.

## Consequences
+ Provider swaps (Groq→multi-provider router in M2) are adapter-local
+ Domain logic testable with in-memory adapters (pytest, cov ≥70%)
− Indirection overhead; adapters must not leak framework types into core

## Alternatives considered
Layered MVC (couples FastAPI to domain); microservices (operational overkill for single-VPS deployment).
