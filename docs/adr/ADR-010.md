# ADR-010 — Behavioral Baseline for Security Anomaly Detection

- **Status:** Accepted
- **Date:** 2026-06-10

## Context
An agentic system with file/message/command access is a high-value target; compromised agents or tokens must be detected, not just prevented.

## Decision
Per-agent and per-node behavioral baselines: rolling stats over tool-call mix, rate, target paths, hours, and payload sizes (stored locally, never uploaded). Deviations beyond threshold (z-score) trigger: log → notify → auto-downgrade trust level (ADR-007) → require re-auth, escalating with severity. Baselines warm up for 7 days before enforcement.

## Consequences
+ Detects token theft / prompt-injection-driven abuse by behavior, not signatures
− False positives during legitimate behavior change; mitigated by warm-up, notify-first escalation, easy user override

## Alternatives considered
Static rate limits only (blind to slow abuse); cloud SIEM (violates sovereignty/privacy stance).
