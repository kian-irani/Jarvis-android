# 📋 گزارش — v54 CF4.2 (اتصالِ MemoryEngine به چت)

> ادامه‌ی CF4. build سبز: `gw :app:compileDebugKotlin :app:testDebugUnitTest`
> → **۲۴۳ تست (۵ جدید) · ۰ failure**. TDD برای `MemoryPolicy` (خالص).

## چه شد
`CloudChatRouter` (تنها قیفِ همه‌ی مسیرهای چت — هم `chat()` و هم `CloudBackend`/orchestrator):
- **read path**: قبل از هر درخواست، `memory.buildContextWindow(message)` به system prompt افزوده می‌شود (guarded؛ "" وقتی خاطره‌ای نیست یا مدلِ embedding دانلود نشده).
- **write path**: بعد از پاسخِ موفق، پیامِ کاربر fire-and-forget روی scopeِ داخلی ذخیره می‌شود **اگر** `MemoryPolicy.worthRemembering` باشد (≥۲۰ کاراکتر و ≥۴ کلمه → «سلام»/کامند/URLِ تک‌کلمه‌ای ذخیره نمی‌شود)، به‌صورتِ `MemoryType.EPISODIC` با importance=0.4.
- `test()` (اعتبارسنجیِ توکن) دست‌نخورده ماند — بدونِ recall/populate (extraSystem پیش‌فرض "").
- `close()` حالا scope را cancel می‌کند.

## TDD
`core/memory/MemoryPolicy` خالص + ۵ تست (blank/short→false، command→false، جمله‌ی باکیفیتِ EN/FA→true، URL/کم‌کلمه→false). تست اول→red→impl→green.

## نکته‌ی واقعیت
فعال‌سازیِ واقعی نیازِ **مدلِ embedding (LM2/MiniLM) روی دستگاه** دارد؛ تا آن دانلود نشود
`buildContextWindow`/`remember` به‌صورتِ graceful no-op هستند (در sandbox هم همین). پس
رفتارِ فعلیِ APK تغییرِ محسوس ندارد مگر مدل دانلود شده باشد — آن‌گاه حافظه‌ی بلندمدتِ
معناییِ فراتر از پنجره‌ی ۶-نوبتیِ `ChatHistoryStore` فعال می‌شود.

## بعدی
